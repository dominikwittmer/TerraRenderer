# TerraRenderer v1.00 – Generated Asset Integration

This sprint connects the renderer directly to the maps produced by `TerraRenderer.AssetBuilder`.

## New generated maps

When an ETOPO GeoTIFF is configured, the builder now creates:

- `earth_height.png`
- `earth_normal.png`
- `earth_watermask.png`
- `earth_curvature.png`
- `earth_ambient_occlusion.png`
- `earth_height_16bit.pgm`

The existing albedo, material, night-emission and night-bloom maps are retained.

## Renderer integration

The renderer can now use:

- tangent-space normal maps instead of estimating relief from the albedo;
- the packed material map (`R=water`, `G=vegetation`, `B=desert`, `A=ice`);
- precomputed ambient occlusion;
- precomputed night bloom.

All generated maps remain optional. Missing files automatically fall back to the previous procedural behavior.

## Configurations

- `config/terrarenderer.json` keeps the original day/night textures and opportunistically loads generated supporting maps.
- `config/terrarenderer.generated.json` uses the complete generated asset set and should be used after running the AssetBuilder.

## Diagnostics

The AssetBuilder writes `diagnostics_contact_sheet.png`, showing every generated map on one sheet. This makes channel and orientation errors easier to identify before rendering the globe.
