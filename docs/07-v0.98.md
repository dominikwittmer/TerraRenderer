# TerraRenderer v0.98 – Camera & Composition

This sprint replaces the default orthographic globe view with a genuine pinhole camera.

## Main changes

- new `PerspectiveEarthProjection` with ray/sphere intersection
- geographic camera target at 37° N, 15° E for a Europe–Mediterranean–Africa hero composition
- configurable camera distance, vertical field of view and roll per layout
- per-pixel view vectors for correct Fresnel and ocean specular response
- perspective-aware atmosphere silhouette and surface haze
- backwards-compatible orthographic layout for A/B comparisons

## Default camera

```json
{
  "projection": "perspective",
  "centerLatitude": 37.0,
  "centerLongitude": 15.0,
  "cameraDistance": 6.0,
  "fieldOfViewDegrees": 20.0,
  "cameraRollDegrees": 0.0
}
```

The camera looks through the geographic target toward the centre of the Earth. `CameraDistance`
controls perspective strength and globe size together; smaller values make the globe larger and
increase perspective. `FieldOfViewDegrees` controls the lens angle.

## Comparison

Render the perspective and orthographic layouts separately:

```powershell
dotnet run --project .\src\TerraRenderer.Cli\TerraRenderer.Cli.csproj -- --layout centered
dotnet run --project .\src\TerraRenderer.Cli\TerraRenderer.Cli.csproj -- --layout orthographic
```
