# TerraRenderer v0.99 – AssetBuilder

v0.99 introduces a reproducible asset pipeline. The renderer no longer has to derive every material property from the visible day texture at runtime.

## Inputs

- Equirectangular day albedo (JPEG/PNG)
- Equirectangular night-light image (JPEG/PNG)
- Optional single-band ETOPO GeoTIFF (signed 16-bit or 32-bit integer, or 32-bit float; stripped layout)

All input maps must use the standard equirectangular convention: longitude -180°..+180° from left to right and latitude +90°..-90° from top to bottom.

## Outputs

- `earth_albedo.jpg`
- `earth_night_intensity.png`
- `earth_night_emission.png`
- `earth_night_bloom.png`
- `earth_material.png` (`R=ocean`, `G=vegetation`, `B=desert`, `A=ice`)
- `earth_height.png` (8-bit normalized preview, when ETOPO is present)
- `earth_height_16bit.pgm` (16-bit normalized source for future processing)
- `earth_watermask.png`
- `earth_normal.png`
- `manifest.json` with dimensions, file sizes and SHA-256 hashes

## Running

```powershell
dotnet run --project .\src\TerraRenderer.AssetBuilder\TerraRenderer.AssetBuilder.csproj -- --config .\config\assetbuilder.json
```

Without ETOPO the builder still generates albedo, night and heuristic material assets. Add the ETOPO path to `sources.elevationGeoTiff` to generate height-derived maps.

## Recommended ETOPO source

Use ETOPO 2022 30 arc-second Bedrock, GeoTIFF. If the downloaded TIFF is tiled, convert it once to a stripped, uncompressed or LZW-compressed single-band TIFF using GDAL:

```powershell
gdal_translate -of GTiff -co TILED=NO -co COMPRESS=LZW ETOPO_2022_v1_30s_N90W180_bed.tif etopo30_bedrock_stripped.tif
```
