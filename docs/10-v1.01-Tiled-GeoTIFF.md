# v1.01 – Direct tiled GeoTIFF import

The AssetBuilder now reads tiled and striped single-band GeoTIFF elevation sources directly.

## ETOPO 2022 support

Validated input profile:

- 43200 × 21600
- 32-bit IEEE floating point
- Adobe Deflate compression
- floating-point predictor
- 256 × 256 tiles
- WGS 84 / EGM2008 height
- NoData -99999

The source is sampled directly into the configured output resolution. The complete 3.7 GB uncompressed elevation raster is never allocated in memory.

## Memory model

For tiled files, an LRU tile cache is used. `height.tileCacheCapacity` controls the maximum number of decompressed source tiles held in memory. The default of 384 is suitable for ETOPO's 256 × 256 Float32 tiles and typically uses about 100 MB for tile data.

For striped files, only a few scanlines are cached.

## Output manifest

`manifest.json` now records source dimensions, tiling, block size, sample format and NoData value.
